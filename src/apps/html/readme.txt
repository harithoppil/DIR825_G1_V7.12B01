本目录所有的文件都是页面的一部分构成。在改动前请阅读下本文件，会更有利于你的修改工作。

本目录的结构如下：

│
html
│
├──skins           包含的文件是编译前的页面风格文件
│   ├─blue         默认风格
│   └─...          其他风格
│  
├──standard        页面的主目录
│   ├─index.html   主页面
│   ├─main.html    登录页面
│   ├─skin         编译后的页面风格
│   ├─status       包含了所有状态页面
│   ├─setup        基本配置页面
│   ├─advanced     高级配置页面
│   ├─management   管理配置页面
│   ├─wireless     无线配置页面
│   ├─help         帮助指南的页面
│   ├─js           通用JS库
│   └─languages    包含各种语言的文件（如中文、英文等）
│      ├─zh_cn     中文语言
│      ├─en_us     英文语言
│      └─...       其他语言

常见问题：

1、如何修改页面的风格？

   页面的风格是由编译开关所决定。从上图的目录结构可知，当你在menuconfig里选中某一种风格时，编译过程中就会将你所选中的风格文件拷贝standard->skin下面，这就实现了风格的客制。当然，在这之前，你应该先制作好你想要的风格文件并将其放到一个你命名好的新文件夹里面，同时，还要在config.in/makefile里边加上你新添加的风格文件夹选项。
    对不同复杂程度的风格制作，取决于项目本身的需要。
    另外，更换了风格后，记得要更新的版本号，以防止浏览器的缓存。

2、如何增加新的语言文件？

   从上图目录可知，增加的语言文件夹应该放到standard->languages下面。新语言文件夹的目录结构必须要和其他语言夹(如中文)的一致，而且每个语言文件的ID也必须和语言内容一一对应。完成语言文件的制作后，需要在config.in里边增加你添加的语言文件选项。

3、如果需要修改某个模块的页面，怎么办？

   找到该模块的页面直接进行修改即可。模块页面与编译无关。但如果你要在菜单去掉某个模块的话，也只需要在menuconfig的编译开关上去勾选即可。
